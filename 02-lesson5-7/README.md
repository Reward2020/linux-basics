# Довідники, права доступу та операція з файлами

## 1. Миттєва довідка через параметр `--help`

Більшість команд і програм у Linux підтримують параметр `--help`, який виводить коротку інструкцію по використанню команди.

### Приклади:

```bash
ls --help
cp --help
grep --help
```

Це швидкий спосіб дізнатися, які опції підтримує команда.

---

### Особливість:

- `--help` працює з **зовнішніми** програмами, які запускаються як окремі файли.
- Не всі **вбудовані команди shell** (bash built-ins) підтримують `--help`.
  
---

## 2. Читання виводу `--help`

- Вивід зазвичай містить опис команди, синтаксис, список опцій.
- Часто досить прокрутити вивід через `less` або `more`, якщо він великий:

```bash
ls --help | less
```

---

## 3. Чому `cd --help` не працює?

- `cd` — це **вбудована команда shell** (built-in), а не окрема програма.
- Вбудовані команди часто не підтримують параметри `--help`.
- Тому `cd --help` виводить помилку або нічого не показує.

---

## 4. Як отримати довідку по вбудованих командах?

### 4.1 Використання команди `help`

Для Bash built-in команд є команда `help`, яка виводить коротку довідку.

```bash
help cd
help echo
help pwd
```

Це офіційна і швидка довідка по вбудованих командах.

---

### 4.2 Використання `man` (manual)

- Для більшості зовнішніх команд доступна детальна документація у вигляді man-сторінок:

```bash
man ls
man cp
man grep
```

- Для вбудованих команд man-сторінки можуть бути відсутні або неповні.

---

### 4.3 `man bash` — довідка по shell

- Для вбудованих команд, які не мають окремої man-сторінки, можна почитати розділ в документації bash:

```bash
man bash
```

- У цьому великому документі є розділ про вбудовані команди.

---

## 5. Інші способи отримати довідку

- `info` — ще один формат документації, більш структурований, ніж man:

```bash
info ls
info bash
```

- Використання `whatis` і `apropos` для пошуку коротких описів:

```bash
whatis ls
apropos copy
```

---

## Підсумок

| Спосіб                    | Що показує                         | Підходить для                  |
|---------------------------|-----------------------------------|-------------------------------|
| `command --help`           | Коротка довідка                   | Зовнішні команди              |
| `help command`             | Довідка по вбудованих командах   | Bash built-in команди         |
| `man command`              | Детальна документація             | Зовнішні команди (іноді built-in) |
| `man bash`                 | Документація по shell і built-in | Вбудовані команди             |
| `info command`             | Інша документація, часто детальніша | Зовнішні команди              |
| `whatis command`           | Короткий опис                    | Пошук команд                  |
| `apropos keyword`          | Пошук по ключовому слову         | Пошук команд і їх описів      |

---

## Приклади

```bash
ls --help          # миттєва довідка по ls
help cd            # довідка по вбудованій команді cd
man grep           # детальна документація по grep
info bash          # інформація по shell bash
whatis mkdir       # короткий опис mkdir
apropos directory  # пошук команд, пов’язаних з "directory"
```


## Права доступу у Linux: що це таке?

Кожен файл або директорія у Linux має набір **прав доступу**, які визначають, хто і що може робити з цим файлом/папкою.

---

## 1. Формат відображення прав доступу (наприклад: `drwxr-xr-x`)

Розглянемо приклад:

```
drwxr-xr-x  2 user group 4096 Jun 10 12:00 myfolder
-rw-r--r--  1 user group 1024 Jun 10 12:00 myfile.txt
```

### Що означають символи `drwxr-xr-x`?

- **Перший символ** — тип файлу:  
  - `d` — директорія (folder)  
  - `-` — звичайний файл  
  - `l` — символічне посилання (link)  
  - інші: `c` (character device), `b` (block device), `s` (socket), `p` (pipe)

- **Наступні 9 символів** — права доступу, розбиті на 3 групи по 3 символи:

| Позиція | Значення | Пояснення         |
|---------|----------|-------------------|
| 2-4     | `rwx`    | права власника (owner) |
| 5-7     | `r-x`    | права групи (group)    |
| 8-10    | `r-x`    | права інших (others)   |

- `r` — право читання (read)  
- `w` — право запису (write)  
- `x` — право виконання (execute)  
- `-` — відсутність відповідного права

---

### Приклад `drwxr-xr-x`:

- `d` — це директорія.
- Власник має права: `rwx` (читати, писати, виконувати).
- Група має права: `r-x` (читати, виконувати, але не писати).
- Інші мають права: `r-x` (читати, виконувати, але не писати).

---

## 2. Як права представлені у числовій (октавній) формі?

Кожне право має числове значення:

| Права         | Значення |
|---------------|----------|
| read (r)      | 4        |
| write (w)     | 2        |
| execute (x)   | 1        |

Щоб отримати число для кожної групи, просто складаємо значення прав:

- `rwx` = 4 + 2 + 1 = 7  
- `rw-` = 4 + 2 + 0 = 6  
- `r-x` = 4 + 0 + 1 = 5  
- `r--` = 4 + 0 + 0 = 4  
- `--x` = 0 + 0 + 1 = 1  
- `---` = 0 + 0 + 0 = 0

---

### Приклад

`drwxr-xr-x` = `d` + `7 5 5`

- Власник: 7 (rwx)
- Група: 5 (r-x)
- Інші: 5 (r-x)

---

## 3. Звідки беруться права?

- При створенні файлу або директорії система призначає права за замовчуванням.
- Ці права залежать від **маски umask** (user file creation mask).
- Umask — це набір бітів, які "забирають" права у стандартних значень.

---

## 4. Що таке umask і як він впливає?

- Стандартні права для створення файлу — `666` (rw-rw-rw-)  
- Стандартні права для створення директорії — `777` (rwxrwxrwx)

**Umask** віднімається від цих стандартних прав.

### Приклад:

- Якщо umask = `022`, то права будуть:
  - Файл: 666 - 022 = 644 (rw-r--r--)
  - Директорія: 777 - 022 = 755 (rwxr-xr-x)

---

## 5. Як переглянути та змінити umask?

- Переглянути:

```bash
umask
```

- Змінити (тимчасово для сесії):

```bash
umask 027
```

- Щоб зробити зміну постійною, додайте команду `umask 027` у свій файл конфігурації shell, наприклад, `~/.bashrc` або `~/.profile`.

---

## 6. Як змінити права вручну?

- Команда `chmod` (change mode) — змінює права.

### Приклади:

- Встановити права `755` (rwxr-xr-x) для файлу:

```bash
chmod 755 filename
```

- Додати право виконання для власника:

```bash
chmod u+x filename
```

- Зняти право запису для групи:

```bash
chmod g-w filename
```

---

## Підсумок

| Поняття          | Опис                                                |
|------------------|-----------------------------------------------------|
| `drwxr-xr-x`     | Формат прав: тип + права власника, групи, інших     |
| `r` (4)          | Право читання                                       |
| `w` (2)          | Право запису                                       |
| `x` (1)          | Право виконання                                    |
| Umask            | Маска, що "віднімає" права при створенні файлу/папки |
| `chmod`          | Команда для зміни прав вручну                       |

## Розробник у Linux-середовищі 
Найчастіше працюватиме з такими каталогами:

- **`/home`** — тут розташовані домашні каталоги користувачів, включно з його власним робочим простором, кодом, налаштуваннями.
- **`/usr`** — тут знаходяться більшість користувацьких програм, бібліотек і документації. Розробник може встановлювати програми, користуватися бібліотеками, дивитися документацію.
- **`/etc`** — каталог з конфігураційними файлами системи та сервісів. Розробник може змінювати конфігурації, наприклад, для серверів, баз даних, середовищ розробки.
- **`/var`** — містить логи, тимчасові та змінні дані, які можуть бути корисні для налагодження програм і сервісів.
- **`/tmp`** — тимчасові файли, іноді використовуються для кешування або проміжних даних під час розробки.
- **`/opt`** — для встановлення додаткових пакетів і програм, не включених у стандартну систему.
- **`/bin` і `/sbin`** — тут знаходяться базові системні утиліти, які часто використовуються у скриптах і робочих процесах.
- **`/lib` і `/usr/lib`** — бібліотеки, які використовуються при розробці та запуску програм.

---

### Висновок

**Основні каталоги для розробника:**

| Каталог | Призначення                            |
|---------|--------------------------------------|
| `/home` | Особисті файли, код, налаштування    |
| `/usr`  | Програми, бібліотеки, документація   |
| `/etc`  | Конфігураційні файли                  |
| `/var`  | Логи, змінні дані                     |
| `/tmp`  | Тимчасові файли                       |
| `/opt`  | Додаткові сторонні програми           |

Ці каталоги — основа для повсякденної роботи розробника в Linux.



# Розглянемо основні операції з файлами.
Також з каталогами та посиланнями в Linux: створення, копіювання, пошук.

## 1. Файли

### 1.1 Створення файлів

- **Команда `touch`** — створює порожній файл або оновлює час модифікації існуючого.

```bash
touch file.txt
```

*Приклад:* створити порожній файл `file.txt`.

- **Команда `echo`** — створює файл з текстом.

```bash
echo "Привіт, світ!" > hello.txt
```

*Приклад:* створити файл `hello.txt` із текстом "Привіт, світ!".

- **Команда `cat`** — створення файлу через введення тексту (натисніть Ctrl+D для завершення).

```bash
cat > notes.txt
Це мої нотатки.
Ctrl+D
```

---

### 1.2 Копіювання файлів

- **Команда `cp`** — копіює файл.

```bash
cp file.txt copy_file.txt
```

*Приклад:* копіюємо `file.txt` у `copy_file.txt`.

- Копіювання з перезаписом (за замовчуванням) або з підтвердженням:

```bash
cp -i file.txt copy_file.txt
```

Опція `-i` запитує підтвердження перед перезаписом.

---

## 2. Каталоги

### 2.1 Створення каталогів

- **Команда `mkdir`** — створює каталог.

```bash
mkdir my_folder
```

*Приклад:* створити каталог `my_folder`.

- Створення вкладених каталогів:

```bash
mkdir -p dir1/dir2/dir3
```

Опція `-p` створює всі проміжні каталоги, якщо їх немає.

---

### 2.2 Копіювання каталогів

- Копіювання каталогу з усім вмістом:

```bash
cp -r my_folder my_folder_copy
```

Опція `-r` (рекурсивно) копіює каталог разом з файлами і підкаталогами.

---

### 2.3 Видалення каталогів

- Видалення порожнього каталогу:

```bash
rmdir my_folder
```

- Видалення каталогу з усім вмістом (обережно!):

```bash
rm -r my_folder
```

---

## 3. Посилання (links)

В Linux є два типи посилань: **жорсткі (hard links)** і **символічні (soft links або symlinks)**.

### 3.1 Створення жорстких посилань

```bash
ln original_file.txt hardlink_file.txt
```

*Пояснення:* `hardlink_file.txt` — це ще одне ім'я для `original_file.txt`. Обидва файли вказують на один і той же inode (фізичний файл).

---

### 3.2 Створення символічних посилань

```bash
ln -s /шлях/до/original_file.txt symlink_file.txt
```

*Пояснення:* `symlink_file.txt` — це спеціальний файл, що містить посилання на `original_file.txt`. Якщо оригінал видалити, посилання стане "битим".

---

## 4. Пошук файлів та каталогів

### 4.1 Пошук за іменем

- Команда `find` — найпотужніший інструмент.

```bash
find /home/user -name "file.txt"
```

*Приклад:* пошук файлу `file.txt` у каталозі `/home/user` і підкаталогах.

---

### 4.2 Пошук за частковим іменем (маскою)

```bash
find /var/log -name "*.log"
```

*Приклад:* знайти всі файли з розширенням `.log` у `/var/log`.

---

### 4.3 Пошук за типом файлу

- Шукати тільки каталоги:

```bash
find /tmp -type d -name "cache"
```

- Шукати тільки файли:

```bash
find /etc -type f -name "*.conf"
```

---

### 4.4 Пошук за допомогою `locate`

Команда `locate` шукає файли за індексом (швидко, але індекс треба оновлювати командою `updatedb`).

```bash
locate file.txt
```

| Створити файл         | `touch`                      | `touch file.txt`                      | Створити порожній файл             |
| Створити файл з текстом| `echo`                       | `echo "text" > file.txt`              | Записати текст у файл              |
| Копіювати файл        | `cp`                         | `cp file.txt copy.txt`                 | Копіювати файл                    |
| Створити каталог      | `mkdir`                      | `mkdir my_folder`                      | Створити каталог                   |
| Копіювати каталог     | `cp -r`                      | `cp -r my_folder copy_folder`          | Рекурсивно копіювати каталог      |
| Створити жорстке посилання | `ln`                    | `ln original.txt hardlink.txt`         | Створити жорстке посилання        |
| Створити символічне посилання | `ln -s`               | `ln -s /path/original.txt symlink.txt` | Створити символічне посилання     |
| Пошук файлу           | `find`                       | `find /home/user -name "file.txt"`    | Знайти файл за іменем             |
| Пошук по масці        | `find`                       | `find /var/log -name "*.log"`          | Знайти файли за шаблоном          |
| Швидкий пошук         | `locate`                     | `locate file.txt`                      | Швидкий пошук за індексом         |
