# Копіювання файлів між машинами (scp, rsync, sftp)

---

## Вступ: Стратегія вибору інструмента

Перед тим, як почати копіювати файли між локальною машиною та сервером, важливо розуміти цілі та вимоги вашої задачі. Від цього залежить вибір інструменту.

| Задача                                | Рекомендований інструмент | Коментар                                     |
|-------------------------------------|---------------------------|----------------------------------------------|
| Швидке копіювання одного або кількох файлів | `scp`                     | Простий, швидкий, мінімальні налаштування   |
| Синхронізація великих директорій з мінімальним трафіком | `rsync`                   | Передає лише змінені частини, підтримує фільтри |
| Інтерактивне керування файлами на віддаленому сервері | `sftp`                    | Працює як FTP, але поверх SSH, зручний для навігації, завантаження та завантаження файлів |

---

## Типові пастки розробника

- Використання `scp` для великих директорій або частих деплоїв — неефективно, призводить до зайвого трафіку.
- Неправильне розуміння шляху з `/` в кінці при `rsync` — може скопіювати не те, що потрібно.
- Забуті права доступу та власники при копіюванні — важливо для коректної роботи застосунку.
- Ігнорування `.git`, `node_modules` та інших кешів — призводить до зайвих копіювань.
- Відсутність dry-run перевірок перед деплоєм — ризик втрати даних.

---

# Основна частина

## Як працює передача файлів поверх SSH і чому це безпечніше за звичайний FTP

- **SSH (Secure Shell)** — це протокол для безпечного віддаленого доступу, який шифрує весь трафік, включно з командами, паролями та файлами.
- **FTP** — передає дані у відкритому вигляді, що робить його вразливим до прослуховування та атак "людина посередині".
- Передача файлів через SSH (scp, rsync, sftp) гарантує конфіденційність, цілісність та аутентифікацію.
  
---

## Архітектура передачі даних

- SSH — не лише термінал. Під капотом SSH працює як тунель для передачі різноманітних даних.
- Наприклад, `scp` та `sftp` використовують SSH-сесію для безпечного копіювання файлів.
- `rsync` може працювати через SSH, передаючи лише різниці між файлами.
  
**Приклад:**

```bash
scp file.txt user@server:/path/to/destination/
```

Тут `scp` відкриває SSH-з'єднання, а потім передає файл у зашифрованому вигляді.

---

## Спадковість правил

- SSH-сесії успадковують правила безпеки, налаштування ключів, а також обмеження користувача.
- Якщо у вас налаштовані SSH ключі, вони використовуються автоматично для `scp`, `rsync`, `sftp`.
- Приклад: якщо у вас заборонено парольний вхід у SSH, всі інструменти, що працюють поверх SSH, також не дозволять підключення без ключа.

---

## Механіка копіювання

Припустимо, у вас є файл `config.env` локально, і ви хочете скопіювати його на сервер.

```bash
echo "API_KEY=12345" > config.env
scp config.env user@server:/home/user/app/
```

- Створюємо файл локально.
- Підключаємося по SSH і передаємо файл.
- На сервері файл з’являється у вказаній директорії.

---

## Чому це важливо для розробника?

- Розробники часто копіюють конфігурації, скрипти, бінарні файли на сервери для тестування чи деплою.
- Вміння швидко і безпечно передавати файли — основа ефективної роботи.
- Автоматизація копіювання — ключ до CI/CD процесів.

---

## Підготовка до роботи

- Наявність SSH доступу до сервера.
- Налаштовані SSH ключі (рекомендується).
- Відкритий порт 22 на сервері (або інший, якщо змінений).
- Відповідні права доступу на сервері для запису у потрібну директорію.

---

# Утиліта `scp` (Secure Copy)

## Філософія scp

- Простий інструмент для швидкого копіювання файлів між локальною машиною та сервером.
- Використовує SSH для безпечної передачі.
- Ідеально підходить для поодиноких файлів або невеликих обсягів.

---

## Базовий синтаксис

```bash
scp [опції] джерело призначення
```

**Приклади:**

- Копіювання локального файлу на сервер:

```bash
scp file.txt user@server:/home/user/
```

- Копіювання файлу з сервера на локальний комп’ютер:

```bash
scp user@server:/home/user/file.txt .
```

---

## Копіювання конфігів (.env)

Приклад `.env` файлу:

```
DB_HOST=localhost
DB_USER=root
DB_PASS=secret
```

Копіюємо на сервер для деплою:

```bash
scp .env user@server:/home/user/app/
```

---

## Копіювання з сервера (Download)

- **Ubuntu 24.04 (Linux):**

```bash
scp user@server:/home/user/app/log.txt ~/Downloads/
```

- **Windows (PowerShell):**

```powershell
scp user@server:/home/user/app/log.txt C:\Users\YourName\Downloads\
```

(Потрібно мати встановлений `scp` — наприклад, через Git Bash або Windows OpenSSH)

---

## Робота з папками (-r)

Копіювання директорії рекурсивно:

```bash
scp -r /local/folder user@server:/remote/folder
```

**Корисні ключі:**

- `-r` — рекурсивне копіювання папок
- `-P PORT` — вказати нестандартний порт SSH
- `-C` — стиснення даних під час копіювання

---

## Обмеження scp

- Не підтримує диференційне копіювання — копіює весь файл цілком.
- Не має гнучких опцій фільтрації.
- Менш ефективний при копіюванні великих директорій.
- Може бути повільнішим через відсутність оптимізації.

---

# Утиліта `rsync`

## Чому rsync кращий за scp?

- Копіює лише змінені частини файлів.
- Підтримує синхронізацію директорій.
- Можна виключати файли/папки.
- Підтримує режим архіву, зберігає права, власників, часові позначки.

---

## Режим архіву (-a)

```bash
rsync -a source/ user@server:/destination/
```

- `-a` = `-rlptgoD` (рекурсивно, зберігає права, власників, часові позначки, симлінки тощо)

---

## Магія фінального слеша (/)

- Якщо вказати `source/` — копіюються **вміст папки**, без самої папки.
- Якщо `source` (без слеша) — копіюється вся папка разом із вмістом.

**Приклад:**

```bash
rsync -a /local/dir/ user@server:/remote/dir/
```

Копіює вміст `dir` у `/remote/dir/`.

```bash
rsync -a /local/dir user@server:/remote/dir/
```

Копіює папку `dir` всередину `/remote/dir/`, тобто буде `/remote/dir/dir`.

---

## Деплой: ігнорування зайвого

Щоб не копіювати зайві файли:

```bash
rsync -a --exclude='.git/' --exclude='node_modules/' /local/app/ user@server:/remote/app/
```

---

## Dry-run: безпека понад усе

Перевірка без реального копіювання:

```bash
rsync -a --dry-run /local/app/ user@server:/remote/app/
```

Виведе, що буде зроблено, без змін.

---

## Видалення зайвого (--delete)

Щоб видалити на сервері файли, яких немає локально:

```bash
rsync -a --delete /local/app/ user@server:/remote/app/
```

---

# `sftp` (Secure FTP)

## Концепція sftp

- Інтерактивний FTP-подібний клієнт поверх SSH.
- Дозволяє керувати файлами на сервері: перегляд, завантаження, завантаження, видалення.
- Зручний для ручного управління файлами.

---

## Підключення та навігація

```bash
sftp user@server
```

Команди:

- `ls` — список файлів на сервері
- `cd` — зміна директорії на сервері
- `pwd` — показати поточну директорію на сервері

---

## Локальний контекст (префікс l)

Команди з префіксом `l` виконуються локально.

- `lls` — локальний список файлів
- `lcd` — зміна локальної директорії

---

## Команди get та put

- Завантажити файл з сервера:

```bash
get remote_file.txt
```

- Відвантажити файл на сервер:

```bash
put local_file.txt
```

---

## Швидкі правки

- `rename oldname newname` — перейменувати файл на сервері
- `rm filename` — видалити файл на сервері
- `mkdir dirname` — створити папку

---

## Коли sftp — правильний вибір?

- Коли потрібна інтерактивна робота з файлами.
- Коли хочете швидко переглянути і змінити вміст серверної директорії.
- Для простих задач без скриптів.

---

|------------|---------------------------------------------|---------------------------------|----------------------------------|
| `scp`      | Швидке копіювання файлів                     | Простий, швидкий                | Неефективний для великих директорій |

| `sftp`     | Інтерактивне керування файлами                | Зручний для ручної роботи      | Менш зручний для автоматизації    |
