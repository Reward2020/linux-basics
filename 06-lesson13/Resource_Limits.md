
# Урок: Обмеження ресурсів у Linux — nice, renice та ulimit

---

## Вступ

У Linux усі процеси змагаються за ресурси системи — CPU, пам’ять, файлові дескриптори тощо. Щоб керувати цим змаганням і захищати систему від перевантажень, використовують інструменти:

- **nice / renice** — для керування пріоритетами процесів (CPU time),
- **ulimit** — для обмеження ресурсів, які процес може використовувати.

---

# Частина 1. Пріоритети процесів: nice

### Що таке nice?

- Значення **nice** — це "ввічливість" процесу.
- Від -20 (найвищий пріоритет) до 19 (найнижчий).
- За замовчуванням — 0.
- Чим нижче nice, тим більше CPU часу отримає процес.

### Як запускати процес з nice?

```bash
nice -n 10 python heavy_script.py
```

- Тут процес буде "ввічливим" і поступатиметься CPU іншим.

### Обмеження

- Підвищувати nice (робити значення більш позитивним) може будь-хто.
- Знижувати nice (ставити від’ємне) — лише root.

---

# Частина 2. Зміна пріоритету в процесі: renice

### Що таке renice?

- Змінює nice для **вже запущеного** процесу.
- Синтаксис:

```bash
renice -n <значення> -p <PID>
```

### Приклад

```bash
renice -n 15 -p 1234  # знизити пріоритет процесу 1234
sudo renice -n -10 -p 1234  # підвищити пріоритет (потрібен sudo)
```

### Перевірка nice

```bash
ps -o pid,ni,cmd -p 1234
top  # дивитись колонку NI
```

# Частина 3. Обмеження ресурсів: ulimit

### Що таке ulimit?

- Встановлює межі використання ресурсів для процесів поточної сесії.
- Дочірні процеси успадковують ці обмеження.

### Перегляд усіх лімітів

```bash
ulimit -a
```

### Основні ліміти

| Ліміт               | Опис                                         | Прапорець |
|---------------------|----------------------------------------------|-----------|
| Відкриті файли      | Максимальна кількість відкритих файлів       | `-n`      |
| Кількість процесів  | Максимальна кількість процесів користувача   | `-u`      |
| Розмір файлу        | Максимальний розмір файлу, створеного процесом | `-f`      |
| Час CPU             | Максимальний CPU time (секунди)               | `-t`      |
| Віртуальна пам’ять  | Максимальний обсяг пам’яті (KB)               | `-v`      |

---

# Частина 4. Soft і Hard limits

| Тип ліміту | Опис                                                            |
|------------|-----------------------------------------------------------------|
| **Soft**   | Поточний ліміт, який можна змінювати користувачу (до hard)      |
| **Hard**   | Максимальний ліміт, змінити може лише root                      |

### Перегляд soft та hard лімітів (на прикладі відкритих файлів):

```bash
ulimit -Sn  # soft limit
ulimit -Hn  # hard limit
```

### Зміна soft ліміту (якщо не перевищує hard):

```bash
ulimit -Sn 2048
```

# Частина 5. Практичні сценарії

---

### Сценарій 1: Зниження пріоритету важкого процесу

Уявімо, що запускаємо скрипт, який сильно навантажує CPU, але не хочемо, щоб він гальмував інші процеси:

```bash
nice -n 15 python heavy_script.py
```

Перевіряємо пріоритет:

```bash
ps -o pid,ni,cmd -p <PID>
```

Якщо процес уже запущений:

```bash
renice -n 19 -p <PID>
```

---

### Сценарій 2: Обмеження кількості відкритих файлів (для Node.js сервера)

Перевіряємо поточний ліміт:

```bash
ulimit -n
```

Задаємо більше (якщо дозволено):

```bash
ulimit -n 4096
```

Запускаємо сервер Node.js.

---

### Сценарій 3: Захист від fork bomb

Обмежуємо кількість процесів користувача:

```bash
ulimit -u 100
```

Тепер користувач не зможе створити більше 100 процесів одночасно.

# Частина 6. Практика: Крок за кроком

1. **Запуск процесу з nice**

```bash
nice -n 10 sleep 300 &
```

2. **Перевірка пріоритету**

```bash
ps -o pid,ni,cmd -p <PID>
```

3. **Зміна пріоритету renice**

```bash
renice -n 0 -p <PID>
```

4. **Перегляд ulimit**

```bash
ulimit -a
```

5. **Зміна soft ліміту відкритих файлів**

```bash
ulimit -Sn 2048
```

# Додаткові поради

- Для постійного встановлення лімітів змінюй конфігураційні файли `/etc/security/limits.conf` (потрібні права root).
- Для запуску процесів з високим пріоритетом — використовуй sudo.
- Використовуй `top` або `htop` для моніторингу пріоритетів (колонка NI).

| Інструмент | Призначення                              | Ключові команди                          |
|------------|-----------------------------------------|-----------------------------------------|
| **nice**  | Запуск процесу з заданим пріоритетом    | `nice -n <значення> <команда>`          |
| **renice**| Зміна пріоритету запущеного процесу     | `renice -n <значення> -p <PID>`         |
| **ulimit**| Обмеження ресурсів процесів              | `ulimit -a`, `ulimit -n`, `ulimit -u`   |
