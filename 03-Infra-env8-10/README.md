# Користувачі, sudo і безпечна робота з правами

---

## 1. Багатокористувацька система Linux

Linux — це багатокористувацька операційна система, де одночасно можуть працювати багато користувачів, кожен зі своїми правами доступу.

- **Кожен користувач має свій унікальний ідентифікатор (UID)**.
- Користувачі можуть входити у систему та працювати у власних домашніх каталогах (`/home/username`).
- Система контролює доступ до файлів, каталогів та ресурсів за допомогою прав власника, групи та інших користувачів.

---

## 2. Хто куди може зайти?

- Користувач може зайти у свій домашній каталог.
- Доступ до інших каталогів чи файлів регулюється правами (читання, запис, виконання).
- Каталог `/root` — домашній каталог суперкористувача (root), доступний лише root.
- Деякі каталоги (наприклад, `/etc`, `/var`) можуть бути доступні для читання або запису лише адміністраторам.

---

## 3. Зміна власника файлу або каталогу

- Власник файлу — це користувач, який має права на цей файл.
- Власник може змінювати права доступу до свого файлу.
- **Навіщо змінювати власника?**

  - Коли файл створюється від імені одного користувача, але потрібно передати його іншому (наприклад, адміністратор передає права на файл розробнику).
  - Для організації правильного доступу до ресурсів.
  - При передачі проектів між користувачами.

- **Команда для зміни власника:**

```bash
sudo chown new_owner file.txt
```

*Приклад:* змінити власника файлу `file.txt` на користувача `ivan`.

```bash
sudo chown ivan file.txt
```

- Зміна власника і групи:

```bash
sudo chown ivan:developers file.txt
```

---

## 4. Передача прав

- Права доступу до файлу визначаються трьома категоріями:

  - Власник (user)
  - Група (group)
  - Інші (others)

- Права бувають:

  - `r` — читання
  - `w` — запис
  - `x` — виконання

- **Команда для зміни прав доступу:**

```bash
chmod [права] файл
```

- Приклад:

```bash
chmod u+rwx,g+rx,o-rwx file.txt
```

- Пояснення: власник отримує повні права, група — читання і виконання, інші — права відбираються.

---

## 5. Звідки беруться права?

- Права встановлюються під час створення файлу.
- За замовчуванням права залежать від **umask** — маски, що віднімається від стандартних прав.

---

## 6. Команда umask

- `umask` — це маска, яка визначає, які права будуть **відсутні** при створенні файлу чи каталогу.
- Типові значення:

  - `022` — файли створюються з правами 644 (rw-r--r--), каталоги — 755 (rwxr-xr-x)
  - `002` — файли 664 (rw-rw-r--), каталоги 775 (rwxrwxr-x)

- Перевірити поточне значення:

```bash
umask
```

- Змінити тимчасово:

```bash
umask 027
```

---

## 7. Sudo та Root

- **Root** — суперкористувач з повними правами у системі.
- Прямий вхід як root часто заборонений або небажаний з міркувань безпеки.
- **`sudo`** — команда, яка дозволяє звичайним користувачам виконувати команди від імені root або іншого користувача, якщо їм надано відповідні права.
- Використання `sudo` фіксується в логах, що підвищує безпеку.

---

### Приклад використання `sudo`:

```bash
sudo apt update
```

Команда оновить списки пакетів від імені root.

---

## 8. Тимчасова сила sudo

- `sudo` дозволяє виконати одну команду з правами root.
- Після введення пароля sudo дозволяє виконувати команди без повторного введення пароля протягом деякого часу (зазвичай 15 хв).
- Щоб вийти з цього "тимчасового режиму", можна використати:

```bash
sudo -k
```

- Щоб відкрити shell з правами root:

```bash
sudo -i
```

# Створення користувача і підготовка робочого середовища

---

## 1. `useradd` проти `adduser`

### `useradd`

- Низькорівнева (сувора) команда для додавання користувача.
- Працює "як є", без інтерактивних підказок.
- Потрібно вручну вказувати багато параметрів, якщо хочете налаштувати користувача детально.
- Не створює домашній каталог за замовчуванням (залежить від параметрів).
- Використовується в скриптах через свою простоту.

**Приклад:**

```bash
sudo useradd -m -s /bin/bash ivan
```

- `-m` — створити домашній каталог.
- `-s` — вказати оболонку.

---

### `adduser`

- Вищого рівня (дбайлива) команда, зазвичай скрипт на Perl чи Python.
- Інтерактивна — запитує пароль, повне ім’я, інші параметри.
- За замовчуванням створює домашній каталог і копіює файли з `/etc/skel`.
- Зручна для швидкого додавання користувача.

**Приклад:**

```bash
sudo adduser ivan
```

- Запустить серію питань (пароль, ім’я тощо).

---

## 2. Як "заселити" нового користувача в готовий дім — `/etc/skel`

- `/etc/skel` — це шаблон домашнього каталогу.
- При створенні домашнього каталогу користувача (опція `-m` у `useradd` або автоматично в `adduser`) вміст `/etc/skel` копіюється у `/home/username`.
- Там можуть бути стандартні конфігураційні файли, наприклад:

```bash
/etc/skel/.bashrc
/etc/skel/.profile
/etc/skel/.bash_logout
```

- Це забезпечує користувачу готове робоче середовище.

---

## 3. Чому одна літера в `usermod` може позбавити вас доступу до груп

- Команда `usermod` використовується для зміни параметрів користувача.
- Параметр `-G` **заміщує** список додаткових груп користувача.
- Якщо ви випадково виконаєте:

```bash
sudo usermod -G group1 username
```

- То всі інші додаткові групи користувача будуть вилучені, і він буде тільки в `group1`.
- Щоб додати групу без вилучення інших, потрібно використовувати `-a` (append) разом з `-G`:

```bash
sudo usermod -a -G group2 username
```

- Якщо забути `-a`, користувач втратить доступ до інших груп, що може призвести до втрати прав доступу.

---

## 4. Магія дефіса в команді `su -`

- Команда `su` — переключитися на іншого користувача.
- `su username` — переключається на користувача, але зберігає поточне середовище (шляхи, змінні).
- `su - username` (або `su -l username`) — запускає логін-сесію користувача, завантажує його середовище як при звичайному вході.

**Чому це важливо?**

- Без дефіса ви можете залишитись з оточенням попереднього користувача, що може призвести до помилок або проблем з правами.
- З дефісом середовище повністю оновлюється — змінні, шляхи, домашній каталог тощо.

---

## 5. Файл `/etc/passwd`

- Основний файл з інформацією про користувачів.
- Формат рядка:

```
username:x:UID:GID:comment:home_directory:shell
```

- Поля:

  - `username` — ім’я користувача.
  - `x` — пароль замінено на `x`, пароль зберігається в `/etc/shadow`.
  - `UID` — унікальний ідентифікатор користувача.
  - `GID` — основна група користувача.
  - `comment` — часто повне ім’я або опис.
  - `home_directory` — шлях до домашнього каталогу.
  - `shell` — оболонка за замовчуванням.

**Приклад:**

```
ivan:x:1001:1001:Ivan Ivanov:/home/ivan:/bin/bash
```

---

## 6. Файл `/etc/shadow`

- Зберігає зашифровані паролі та політику паролів.
- Формат рядка:

```
username:encrypted_password:last_change:min_days:max_days:warn:inactive:expire:reserved
```

- Поля:

  - `username` — ім’я користувача.
  - `encrypted_password` — хеш пароля або спеціальні значення (`*`, `!` для заблокованих).
  - `last_change` — дата останньої зміни пароля (у днях від 1970-01-01).
  - `min_days` — мінімальна кількість днів між змінами пароля.
  - `max_days` — максимальна кількість днів до зміни пароля.
  - `warn` — кількість днів до закінчення пароля, коли користувачу надсилається попередження.
  - `inactive` — кількість днів після закінчення пароля, коли обліковий запис блокується.
  - `expire` — дата закінчення облікового запису.
  - `reserved` — зарезервоване поле.

**Приклад:**

```
ivan:$6$abcd1234$Yt0...:18700:0:99999:7:::
```

---

## 7. В чому різниця `useradd` та `adduser`?

| Параметр              | useradd                         | adduser                         |
|-----------------------|--------------------------------|--------------------------------|
| Рівень                | Низькорівневий інструмент      | Вищого рівня, інтерактивний    |
| Створення домашнього каталогу | Потрібно вказувати `-m`         | Створює автоматично             |
| Копіювання `/etc/skel` | Потрібно явно вказувати          | Робить автоматично              |
| Запит пароля          | Ні (потрібно задавати окремо)   | Так (інтерактивно)              |
| Зручність             | Для скриптів                   | Для користувача                 |

## 8. Як додати користувача в групу?

- Для додавання користувача в додаткову групу:

```bash
sudo usermod -a -G groupname username
```

- Наприклад:

```bash
sudo usermod -a -G sudo ivan
```

- Перевірити групи користувача:

```bash
groups username
```

---

## 9. Адміністрування конкретної групи

- Створити групу:

```bash
sudo groupadd groupname
```

- Видалити групу:

```bash
sudo groupdel groupname
```

- Додати користувача до групи — як показано вище.

- Редагувати файл груп вручну можна через:

```bash
sudo vigr
```

- Формат рядка у `/etc/group`:

```
groupname:x:GID:user1,user2,user3
```

# Керування процесами в Linux

---

## 1. Різниця між програмою на диску і процесом у пам'яті

- **Програма на диску** — це просто набір інструкцій (файл виконуваного коду), який зберігається на носії (жорсткий диск, SSD).
- **Процес у пам'яті** — це запущена програма, яка завантажена в оперативну пам’ять, має свій власний контекст виконання (пам’ять, регістри, PID, відкриті файли тощо).
- Тобто **програма** — це статичний код, а **процес** — це динамічна сутність, що виконується.

---

## 2. Що таке PID і чому він ніколи не повторюється (поки система працює)

- **PID (Process ID)** — унікальний ідентифікатор процесу в системі.
- PID присвоюється кожному процесу при його створенні.
- PID — це ціле число (зазвичай від 1 до 32768 або більше, залежно від системи).
- **Чому PID не повторюється?**

  - Система керує PID, уникаючи повторного використання активних PID.
  - Коли процес завершується, його PID звільняється, але система не відразу присвоює його новому процесу.
  - PID може повторитися, але лише після того, як всі інші PID були використані (циклічно).
  - Поки система працює і PID активно призначаються, повторення PID для одночасно існуючих процесів не буває.

---

## 3. Як зробити моментальний знімок системи за допомогою `ps`

Команда `ps` виводить список процесів.

### Приклад:

```bash
ps aux
```

- `a` — показати процеси всіх користувачів.
- `u` — вивід у зручному форматі з інформацією про користувача.
- `x` — показати процеси, які не прив’язані до терміналу.

**Приклад виводу:**

```
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1 169084  5644 ?        Ss   10:00   0:02 /sbin/init
ivan     2345  0.3  1.2 256000 24500 pts/0    Ss   10:10   0:10 /usr/bin/bash
ivan     2350  5.0  3.0 512000 60000 pts/0    R+   10:11   0:05 top
```

---

## 4. Як читати "приладову панель" `top` та розуміти Load Average

Команда `top` — це інтерактивний монітор ресурсів.

### Основні блоки `top`:

```
top - 10:15:30 up  2:00,  2 users,  load average: 0.15, 0.10, 0.05
Tasks: 150 total,   1 running, 149 sleeping,   0 stopped,   0 zombie
%Cpu(s):  5.0 us,  1.0 sy,  0.0 ni, 93.0 id,  0.5 wa,  0.0 hi,  0.5 si,  0.0 st
KiB Mem :  8000000 total,  2000000 free,  4000000 used,  2000000 buff/cache
KiB Swap:  2000000 total,  2000000 free,        0 used.  3500000 avail Mem 

  PID USER      PR  NI    VIRT    RES    SHR S  %CPU %MEM     TIME+ COMMAND
 2345 ivan      20   0  256000  24500   5000 S   1.3  0.3   0:10.23 bash
 2350 ivan      20   0  512000  60000  10000 R  10.0  0.7   0:05.12 top
```

### Пояснення:

- **Load Average** — це середнє навантаження на систему за останні 1, 5 та 15 хвилин.
  - Відображає середню кількість процесів, які або працюють, або очікують на CPU.
  - Значення близько до кількості ядер CPU означає максимальне навантаження.
  - Наприклад, якщо у вас 4 ядра, load average 4.0 — це повне завантаження.

- **Tasks** — кількість процесів, їх статуси.
- **%Cpu(s)** — розподіл часу CPU (us — користувацький код, sy — системний, id — бездіяльність).
- **Mem** — використання оперативної пам’яті.
- **Swap** — використання swap.

---

## 5. Різниця між ввічливим проханням (SIGTERM) та пострілом в голову (SIGKILL)

- **SIGTERM (signal 15)** — це сигнал "запросити завершення".
  - Процес отримує сигнал і може коректно завершити роботу: закрити файли, звільнити ресурси.
  - Це "ввічливий" спосіб попросити процес завершитись.
- **SIGKILL (signal 9)** — це сигнал "примусово вбити".
  - Процес завершується негайно, без можливості обробити сигнал.
  - Не дозволяє процесу виконати чистку.
  - Використовується, коли процес не реагує на SIGTERM.

---

## 6. Чому `htop` — це любов з першого погляду

- `htop` — це покращена версія `top`.
- Має кольоровий інтерфейс, зручну навігацію.
- Показує дерево процесів.
- Дозволяє легко шукати, фільтрувати, послідовно завершувати процеси (через меню).
- Показує графіки CPU, пам’яті.
- Інтерактивний — можна вибирати процеси стрілками.

```bash
kill 2345
```

- Надсилає SIGTERM процесу з PID 2345.

---

### 4. Вбити процес примусово (SIGKILL)

```bash
kill -9 2345
```

- Надсилає SIGKILL.

---

### 5. Запуск `top`

```bash
top
```

- Інтерактивний монітор системи.

---

### 6. Запуск `htop` (потрібно встановити)

```bash
htop
```

- Зручний монітор з кольоровим інтерфейсом.

# Підсумок

| Тема                         | Коротко                                                   |
|------------------------------|-----------------------------------------------------------|
| Програма vs Процес           | Програма — файл на диску, процес — запущена програма в пам’яті |
| PID                         | Унікальний ідентифікатор процесу, не повторюється поки процес активний |
| `ps`                        | Статичний знімок процесів                                  |
| `top`                       | Динамічний монітор, показує навантаження та ресурси       |
| Load Average                | Середнє навантаження за 1, 5, 15 хвилин                   |
| SIGTERM (15)                | Ввічливий сигнал завершення                                |
| SIGKILL (9)                 | Примусовий сигнал завершення                               |
| `htop`                      | Покращений `top` з кольорами та інтерактивністю           |
