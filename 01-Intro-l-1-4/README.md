# Linux OS

У кореневій директорії Linux (`/`) знаходяться основні системні каталоги, які утворюють структуру файлової системи. Вони містять різні типи файлів і підкаталогів, необхідних для роботи операційної системи.

Типові каталоги в кореневій директорії Linux:

- `/bin` — базові виконувані файли (команди), необхідні для роботи системи в однокористувацькому режимі та для всіх користувачів.
- `/boot` — файли, необхідні для завантаження системи (ядро Linux, initrd, завантажувач GRUB тощо).
- `/dev` — файли пристроїв (спеціальні файли для доступу до апаратного забезпечення).
- `/etc` — конфігураційні файли системи та сервісів.
- `/home` — домашні каталоги користувачів.
- `/lib` — бібліотеки, необхідні для запуску програм у `/bin` і `/sbin`.
- `/media` — точка монтування для знімних носіїв (CD/DVD, флешки).
- `/mnt` — тимчасова точка монтування для файлових систем.
- `/opt` — додаткові пакети та програми.
- `/proc` — віртуальна файлова система з інформацією про процеси та ядро.
- `/root` — домашній каталог суперкористувача (root).
- `/run` — тимчасові файли, що описують поточний стан системи.
- `/sbin` — системні виконувані файли, доступні здебільшого суперкористувачу.
- `/srv` — дані для сервісів, які надає система (наприклад, веб-сервер).
- `/sys` — віртуальна файлова система для взаємодії з ядром.
- `/tmp` — тимчасові файли, які можуть видалятися після перезавантаження.
- `/usr` — більшість користувацьких програм, бібліотек, документації.
- `/var` — змінні дані, логи, пошта, черги друку.

Ця структура є стандартною для більшості дистрибутивів Linux і відповідає стандарту Filesystem Hierarchy Standard (FHS).


---

### Що таке amd64?

- **amd64** — це архітектура 64-бітних процесорів, яка також відома як **x86-64** або **x64**.
- Вона була розроблена компанією AMD як розширення 32-бітної архітектури x86 (яка була популярна на персональних комп’ютерах).
- Сьогодні amd64 — це стандартна 64-бітна архітектура для більшості настільних і серверних процесорів від Intel і AMD.
- Архітектура підтримує 64-бітні адреси пам’яті, що дозволяє використовувати більше оперативної пам’яті та підвищує продуктивність.

---

### Що таке arm64?

- **arm64** (також відомий як **AArch64**) — це 64-бітна архітектура процесорів ARM.
- ARM — це інша сімейство процесорних архітектур, які відрізняються від x86 за принципом побудови (RISC — Reduced Instruction Set Computing).
- ARM-процесори широко використовуються в мобільних пристроях (смартфонах, планшетах), а також у вбудованих системах і все частіше в ноутбуках і серверах.
- arm64 — це 64-бітне розширення ARMv8 архітектури, яке підтримує більший обсяг пам’яті та покращену продуктивність у порівнянні з 32-бітним ARM.

---

### В чому різниця?

| Параметр               | amd64 (x86-64)                        | arm64 (AArch64)                     |
|------------------------|-------------------------------------|-----------------------------------|
| **Тип архітектури**    | CISC (Complex Instruction Set)      | RISC (Reduced Instruction Set)    |
| **Розробник**          | AMD (пізніше Intel)                  | ARM Holdings                      |
| **Застосування**       | ПК, сервери, ноутбуки                | Смартфони, планшети, вбудовані системи, ноутбуки, сервери |
| **Енергоефективність** | Менш енергоефективна                 | Вища енергоефективність           |
| **Інструкції**         | Складніша інструкційна система      | Спрощена інструкційна система     |
| **Сумісність**         | Сумісність з 32-бітними x86 програмами | Зворотна сумісність з 32-бітним ARM |
| **Популярність**       | Домінує в десктопах і серверах      | Домінує в мобільних пристроях, зростає в десктопах і серверах |

---

### Чому це важливо?

- Програми, скомпільовані для amd64, не працюватимуть на arm64 без емуляції або спеціальної версії.
- Вибір архітектури впливає на продуктивність, енергоспоживання і сумісність програмного забезпечення.
- Сучасні операційні системи (Linux, Windows, macOS) підтримують обидві архітектури, але для кожної потрібні свої версії програм.

---

### Підсумок

- **amd64** — це 64-бітна архітектура процесорів сімейства x86 від AMD/Intel, основна для ПК і серверів.
- **arm64** — це 64-бітна архітектура ARM, популярна в мобільних пристроях і все більше в десктопах і серверах.
- Вони відрізняються за архітектурою, сумісністю та сферою застосування.

## Короткий огляд базових команд терміналу Linux/Unix, 
Вони допоможуть вам ефективно працювати з файлами і директоріями:

---

### 1. `ls` — показати вміст директорії

**Призначення:** Виводить список файлів і папок у поточній або вказаній директорії.

**Приклади:**
- `ls` — показати файли в поточній директорії.
- `ls /home/user` — показати файли в директорії `/home/user`.
- `ls -l` — детальний список (права, власник, розмір, дата).
- `ls -a` — показати всі файли, включно з прихованими (які починаються з `.`).

---

### 2. `cd` — змінити директорію

**Призначення:** Перейти в іншу директорію.

**Приклади:**
- `cd /home/user/Documents` — перейти в папку Documents.
- `cd ..` — перейти на рівень вище.
- `cd ~` або просто `cd` — перейти в домашню директорію користувача.
- `cd -` — повернутися в попередню директорію.

---

### 3. `cp` — копіювати файли або директорії

**Призначення:** Копіює файли або папки.

**Приклади:**
- `cp file1.txt file2.txt` — скопіювати файл `file1.txt` у `file2.txt`.
- `cp file.txt /home/user/Documents/` — скопіювати файл у вказану директорію.
- `cp -r folder1 folder2` — рекурсивно скопіювати папку `folder1` у `folder2` (включно з усіма файлами і підпапками).

---

### 4. `mv` — перемістити або перейменувати файли/папки

**Призначення:** Переміщує файл/папку або змінює їм ім’я.

**Приклади:**
- `mv oldname.txt newname.txt` — перейменувати файл.
- `mv file.txt /home/user/Documents/` — перемістити файл у вказану директорію.
- `mv folder1 folder2` — перемістити папку (або перейменувати).

---

### 5. `cat` — вивести вміст файлу

**Призначення:** Виводить вміст текстового файлу у термінал.

**Приклади:**
- `cat file.txt` — показати вміст файлу.
- `cat file1.txt file2.txt` — вивести вміст обох файлів послідовно.
- `cat > newfile.txt` — створити файл і ввести текст (для виходу натисніть Ctrl+D).

---

### 6. `man` — довідкова система (manual)

**Призначення:** Показує документацію по команді.

**Приклади:**
- `man ls` — показати довідку по команді `ls`.
- `man cp` — довідка по `cp`.
- Вийти з довідки — натисніть `q`.

---

## Додаткові поради:

- Для перегляду великих файлів замість `cat` часто використовують `less` або `more`.
- Команди можна комбінувати з опціями для розширення функціоналу.
- Для автодоповнення імен файлів у терміналі натискайте клавішу `Tab`.


## Давайте розберемо, як у Linux подивитися, який **inode** присвоєний файлу.

---

### Що таке inode?

- **inode** — це унікальний ідентифікатор файлу в файловій системі.
- Він містить метадані про файл (власник, права доступу, час створення, розмір і т.д.), але не містить ім'я файлу.
- Ім'я файлу зберігається у каталозі і вказує на відповідний inode.

---

### Як подивитися inode файлу?

Використовують команду `ls` з опцією `-i` (індекс):

```bash
ls -i ім'я_файлу
```

---

### Приклад:

```bash
$ ls -i example.txt
123456 example.txt
```

- Число `123456` — це inode файлу `example.txt`.

---

### Як подивитися inode для всіх файлів у каталозі?

```bash
ls -i
```

Виведе список файлів з їхніми inode.

---

### Додатково: команда `stat`

Команда `stat` показує детальну інформацію про файл, включно з inode:

```bash
stat example.txt
```

Приклад виводу:

```
  File: example.txt
  Size: 1024       Blocks: 8          IO Block: 4096   regular file
Device: 802h/2050d Inode: 123456      Links: 1
Access: 2024-06-01 12:00:00.000000000 +0300
Modify: 2024-06-01 11:50:00.000000000 +0300
Change: 2024-06-01 11:55:00.000000000 +0300
```

Тут `Inode: 123456` — це номер inode.

---

### Підсумок

- Щоб дізнатися inode файлу, виконай:

```bash
ls -i ім'я_файлу
```

- Або для детальної інформації:

```bash
stat ім'я_файлу
```

Команда `find` — це дуже потужний інструмент у Unix/Linux для пошуку файлів і каталогів за різними критеріями та виконання над ними різних операцій.


---

## Основний синтаксис find

```bash
find [шлях] [умови_пошуку] [операції]
```

- **шлях** — директорія, у якій шукаємо (наприклад, `.` — поточна директорія)
- **умови_пошуку** — параметри, за якими шукаємо (ім'я файлу, розмір, дата, права тощо)
- **операції** — що робити з знайденими файлами (вивести, видалити, виконати команду і т.п.)

---

## Приклади використання `find`

### 1. Знайти файл за іменем

```bash
find . -name "file.txt"
```

Шукає файли з іменем `file.txt` у поточній директорії та піддиректоріях.

---

### 2. Знайти файли за шаблоном імені (з урахуванням регістру)

```bash
find /path/to/dir -iname "*.jpg"
```

Шукає всі файли з розширенням `.jpg` або `.JPG` (регістр не має значення).

---

### 3. Знайти файли за розміром

- Більше 10 Мб:

```bash
find . -size +10M
```

- Менше 100 Кб:

```bash
find . -size -100k
```

---

### 4. Знайти файли за датою модифікації

- Змінені за останні 7 днів:

```bash
find . -mtime -7
```

- Змінені більше ніж 30 днів тому:

```bash
find . -mtime +30
```

---

### 5. Знайти порожні файли або каталоги

- Порожні файли:

```bash
find . -type f -empty
```

- Порожні каталоги:

```bash
find . -type d -empty
```

---

### 6. Знайти файли з певними правами доступу

- Файли з правами 755:

```bash
find . -perm 755
```

- Файли, у яких є права на виконання для власника:

```bash
find . -perm -u=x
```

---

### 7. Видалити знайдені файли

**Увага! Будьте обережні з цією командою!**

```bash
find . -name "*.tmp" -delete
```

Видаляє всі файли з розширенням `.tmp`.

Або через `-exec`:

```bash
find . -name "*.tmp" -exec rm {} \;
```

---

### 8. Виконати команду над кожним знайденим файлом

```bash
find . -name "*.log" -exec gzip {} \;
```

Архівує всі `.log` файли за допомогою `gzip`.

---

### 9. Знайти файли, які належать певному користувачу

```bash
find /var -user username
```

---

### 10. Знайти файли за типом

- Знайти всі каталоги:

```bash
find . -type d
```

- Знайти всі символічні посилання:

```bash
find . -type l
```

---

### 11. Комбінування умов

Знайти всі `.txt` файли, які були змінені за останні 10 днів:

```bash
find . -name "*.txt" -mtime -10
```

---

## Коротко про деякі важливі опції

| Опція        | Опис                                 |
|--------------|-------------------------------------|
| `-name`      | Ім'я файлу (з урахуванням регістру) |
| `-iname`     | Ім'я файлу без урахування регістру  |
| `-type`      | Тип об'єкта (f — файл, d — каталог) |
| `-size`      | Розмір файлу                        |
| `-mtime`     | Час останньої модифікації (у днях)  |
| `-user`      | Власник файлу                      |
| `-perm`      | Права доступу                      |
| `-exec`      | Виконати команду над знайденими файлами |
| `-delete`    | Видалити знайдені файли             |


## Абсолютні шляхи

- **Що це?**  
  Абсолютний шлях — це повний шлях від кореневої директорії `/` до потрібного файлу або папки.
- **Особливість:**  
  Починається завжди зі слеша `/`.
- **Приклад:**  
  `/home/user/Documents/file.txt` — абсолютний шлях до файлу `file.txt`.

### Приклади використання:

```bash
cd /home/user/Documents
ls /var/log
cat /etc/passwd
```

---

## Відносні шляхи

- **Що це?**  
  Відносний шлях задається від поточної робочої директорії (де ви зараз знаходитеся).
- **Особливість:**  
  Не починається зі слеша `/`.
- **Спеціальні позначення:**  
  - `.` — поточна директорія  
  - `..` — батьківська директорія (на рівень вище)

### Приклад:

Якщо ви зараз у директорії `/home/user`, і хочете перейти у `/home/user/Documents`:

```bash
cd Documents
```

або перейти на рівень вище:

```bash
cd ..
```

---

## Приклади навігації з абсолютними і відносними шляхами

Припустимо, ваша поточна директорія:

```
/home/user
```

### Абсолютні шляхи

```bash
cd /home/user/Documents
ls /var/log
cat /etc/hosts
```

### Відносні шляхи

```bash
cd Documents          # перейти в Documents від поточної директорії /home/user
ls ../                # показати вміст батьківської директорії (/home)
cat Documents/file.txt  # прочитати файл у Documents відносно поточної директорії
cd ./Documents        # те саме, що просто cd Documents
cd ../../             # піднятися на два рівні вище
```

---

## Пояснення:

- `cd /home/user/Documents` — абсолютний шлях, ігнорує поточну директорію.
- `cd Documents` — відносний шлях, переходимо у папку Documents всередині поточної директорії.
- `cd ..` — піднятися на один рівень вгору.
- `cd ./Documents` — `.` означає поточну директорію, тому це те саме, що `cd Documents`.
- `cd ../../` — піднятися на два рівні вгору.
