
# Лекція: Вступ до systemd та керування сервісами в Linux  


## Вступ

Сьогодні ми дізнаємось, що таке systemd — "бос" усіх процесів у сучасних Linux-системах. Навчимося керувати сервісами, читати логи та налаштовувати автозапуск програм. Це дуже важливо для адміністраторів, DevOps-інженерів і всіх, хто хоче розуміти, як працює сервер.

---

# Частина 1: 👑 Архітектура та «Бос» системи — systemd (PID 1)

---

### Що таке systemd?

- **systemd** — це перший процес, який запускається при старті Linux-системи.  
- Він має PID 1 — це означає, що він "батько" всіх інших процесів.  
- systemd відповідає за запуск, зупинку, моніторинг та управління усіма сервісами і процесами.

---

### Зони відповідальності systemd

- Ініціалізація системи (завантаження сервісів)  
- Керування демонами (службами)  
- Логування подій (через journald)  
- Керування ресурсами (через cgroups)  
- Автоматичне перезапуск сервісів у разі падіння

---

### Що таке демон (Service)?

- Демон — це фоновий процес, який виконує певну задачу (наприклад, веб-сервер, база даних).  
- Демони не взаємодіють безпосередньо з користувачем, а працюють у бекграунді.

---

### Практична перевірка: Хто головний?

Відкрийте термінал і виконайте:

```bash
ps -p 1 -o comm=
```

Відповідь має бути `systemd` — це і є головний процес.

---

### Де це застосовується?

- Кожен сервер чи робоча станція на Linux використовує systemd для управління процесами.  
- Наприклад, коли ви запускаєте сервер Apache або Nginx, systemd керує їх життєвим циклом.

---

# Частина 2: 🎮 Оперативне керування сервісами

Щоб встановити і запустити Nginx на Ubuntu 22.04, виконай такі кроки:

---

### Крок 1: Онови список пакетів

```bash
sudo apt update
```

---

### Крок 2: Встанови Nginx

```bash
sudo apt install nginx -y
```

---

### Крок 3: Запусти Nginx (якщо він не запустився автоматично)

```bash
sudo systemctl start nginx
```

---

### Крок 4: Перевір статус сервісу

```bash
sudo systemctl status nginx
```

Якщо все добре — побачиш, що сервіс активний (active/running).

---

### Крок 5: Дозволь HTTP-трафік через брандмауер (якщо ufw увімкнено)

```bash
sudo ufw allow 'Nginx HTTP'
sudo ufw status
```

---

### Крок 6: Перевір роботу nginx у браузері або через curl

Відкрий у браузері:

```
http://localhost
```

або виконай:

```bash
curl http://localhost
```

Ти маєш побачити стандартну сторінку Nginx із повідомленням "Welcome to nginx!".

---

### systemctl vs service

- `systemctl` — сучасна утиліта для управління systemd-сервісами.  
- `service` — старіша команда, яка працювала з init-системами.  
- Рекомендуємо використовувати `systemctl`.

---

### Базові команди:

- Перевірити статус сервісу:

```bash
systemctl status nginx
```

- Запустити сервіс:

```bash
sudo systemctl start nginx
```

- Зупинити сервіс:

```bash
sudo systemctl stop nginx
```

- Перезапустити сервіс:

```bash
sudo systemctl restart nginx
```

---

### Діагностика падіння сервісу

Якщо сервіс не запускається або падає, подивіться логи:

```bash
journalctl -u nginx.service
```

---

### Перегляд усіх активних сервісів

```bash
systemctl list-units --type=service --state=running
```

---

### Практика

1. Встановіть nginx (або інший сервіс на ваш вибір):

```bash
sudo apt install nginx -y
```

2. Запустіть його:

```bash
sudo systemctl start nginx
```

3. Перевірте статус:

```bash
systemctl status nginx
```

4. Зупиніть сервіс:

```bash
sudo systemctl stop nginx
```

5. Перегляньте логи:

```bash
journalctl -u nginx.service --since "10 minutes ago"
```

---

### Де це застосовується?

- Коли ви адмініструєте сервер, вам потрібно швидко запускати, зупиняти або перевіряти сервіси.  
- Наприклад, після оновлення веб-сервера ви перезапускаєте сервіс.

---

# Частина 3: ⚙️ Автозавантаження сервісів

---

### Навіщо потрібне автозавантаження?

- Щоб після перезавантаження сервера важливі сервіси запускалися автоматично.  
- Це гарантує безперервність роботи.

---

### Команди для автозапуску

- Включити автозапуск:

```bash
sudo systemctl enable nginx
```

- Вимкнути автозапуск:

```bash
sudo systemctl disable nginx
```

- Перевірити, чи ввімкнено автозапуск:

```bash
systemctl is-enabled nginx
```

---

### Практика

1. Увімкніть автозапуск nginx:

```bash
sudo systemctl enable nginx
```

2. Перезавантажте сервер (або емулятор):

```bash
sudo reboot
```

3. Після перезавантаження перевірте статус:

```bash
systemctl status nginx
```

---

### Де це застосовується?

- Веб-сервери, бази даних, черги повідомлень — всі ці сервіси мають автоматично запускатися після ребута.

---

# Частина 4: 🔍 Робота з журналом подій (journald)

---

### Що таке journald?

- Це системний журнал, який збирає логи від усіх сервісів і ядра.  
- Логи структуровані, що полегшує їх аналіз.

---

### Перегляд логів конкретного сервісу

```bash
journalctl -u nginx.service
```

---

### Режим реального часу

```bash
journalctl -u nginx.service -f
```

Це як `tail -f`, показує логи в реальному часі.

---

### Практика

1. Запустіть nginx:

```bash
sudo systemctl start nginx
```

2. Відкрийте інший термінал і запустіть:

```bash
journalctl -u nginx.service -f
```

3. Відкрийте браузер і зайдіть на сервер (http://localhost), ви побачите нові записи в журналі.

---

### Де це застосовується?

- Для швидкого пошуку причин помилок сервісів.  
- Для моніторингу роботи сервісів у реальному часі.

# Частина 5: 🚀 Найкращі практики DevOps

---

### Порядок запуску (Unit-файли)

- systemd використовує **unit-файли** (зазвичай у `/etc/systemd/system/` або `/lib/systemd/system/`) для опису сервісів.  
- В unit-файлі прописано, як запускати сервіс, залежності, ресурси.

---

### Автоматичне відновлення

- Можна налаштувати автоматичний перезапуск сервісу, якщо він впав.  
- Приклад у unit-файлі:
```bash
sudo systemctl edit --full nginx.service
```


```ini
[Service]
Restart=always
RestartSec=5
```

---

### Тестування конфігурації

- Після зміни unit-файлу виконайте:

```bash
sudo systemctl daemon-reload
```

- Потім перезапустіть сервіс.

---

### Шаблонні сервіси

- systemd підтримує шаблони для однотипних сервісів (наприклад, `myapp@instance1.service`, `myapp@instance2.service`).  
- Це зручно для масштабування.

---

### Використання Git для конфігурацій

- Unit-файли та скрипти можна тримати під контролем версій у Git.  
- Це дозволяє відслідковувати зміни і швидко відкотитись.

---

### Практика

1. Створіть свій простий unit-файл:

```ini
sudo nano /etc/systemd/system/myapp.service
```

Вставте:

```ini
[Unit]
Description=My Test App

[Service]
ExecStart=/bin/bash -c 'while true; do echo "Hello from MyApp"; sleep 10; done'
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

2. Завантажте конфігурацію:

```bash
sudo systemctl daemon-reload
```

3. Запустіть сервіс:

```bash
sudo systemctl start myapp
```

4. Перегляньте логи:

```bash
journalctl -u myapp -f
```

5. Увімкніть автозапуск:

```bash
sudo systemctl enable myapp
```

## Як зробити, щоб можна було запускати `systemctl restart nginx` без пароля?

### Використовувати `sudo` без пароля для конкретної команди

1. Відкрий файл sudoers через `visudo` (щоб уникнути помилок):

```bash
sudo visudo
```

2. Додай в кінець (заміни `goit` на свій логін):

```bash
goit ALL=NOPASSWD: /bin/systemctl restart nginx.service, /bin/systemctl start nginx.service, /bin/systemctl stop nginx.service
```

3. Тепер можна запускати:

```bash
sudo systemctl restart nginx
```

без запиту пароля.

- Для власних додатків і сервісів, щоб забезпечити їх надійність і легке управління.  
- DevOps-команди використовують такі налаштування для автоматизації.

# Підсумок

- **systemd** — це центральний менеджер процесів у Linux.  
- **systemctl** — інструмент для керування сервісами.  
- **journalctl** — для читання логів.  
- Автоматичний запуск, відновлення і шаблонні сервіси — основа стабільної роботи серверів.  
- Практичні навички допоможуть вам впевнено керувати Linux-серверами.
