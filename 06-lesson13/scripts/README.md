## Пояснення скрипта `heavy_script.py`

- Цикл робить 1 мільйон простих операцій, що дає навантаження на CPU.
- Після цього робить паузу 0.1 секунди, щоб система не була повністю завантажена.
- Такий скрипт створює помірне навантаження для демонстрації.

---

## Запуск з nice

```bash
nice -n 10 python3 heavy_script.py
```

---

## Перевірка пріоритету процесу

Запускаємо у іншому терміналі:

```bash
ps -o pid,ni,cmd -C python3
```

або

```bash
top
```

і дивимось колонку NI (nice value).

## Як встановити Python

### Перевірка, чи встановлений Python

Відкрийте термінал і введіть:

```bash
python3 --version
```

Якщо побачите версію (наприклад, `Python 3.8.10`), Python вже встановлений.

---

### Встановлення Python (якщо не встановлений)

#### На Ubuntu/Debian:

```bash
sudo apt update
sudo apt install python3
```

#### На CentOS/Fedora:

```bash
sudo dnf install python3
```

або

```bash
sudo yum install python3
```

---

## Як запустити скрипт

Припустимо, файл називається `heavy_script.py`.

```bash
python3 heavy_script.py
```

---

## Як зупинити скрипт

1. Якщо скрипт запущений у вашому терміналі, просто натисніть:

```
Ctrl + C
```

Це надішле сигнал переривання і зупинить скрипт.

2. Якщо скрипт запущений у фоновому режимі (наприклад, з `&` в кінці):

- Знайдіть PID процесу:

```bash
ps aux | grep heavy_script.py
```

- Зупиніть процес командою:

```bash
kill <PID>
```

або більш "жорстко":

```bash
kill -9 <PID>
```


# Приклади роботи з `ulimit` / open_files_test.sh

---

## 1. Перегляд поточних лімітів

```bash
ulimit -a
```

Виведе всі поточні обмеження для вашої сесії.

---

## 2. Обмеження кількості відкритих файлів

### Перевірка поточного ліміту

```bash
ulimit -n
```

### Зміна soft-ліміту (максимум — hard limit)

```bash
ulimit -Sn 50
```

Обмежує кількість відкритих файлів до 50.

### Перевірка

Спробуйте відкрити багато файлів одночасно (в іншому терміналі або в скрипті).

---

### Приклад скрипта для перевірки відкритих файлів
- Дайте права на виконання:

```bash
chmod +x open_files_test.sh
```

- Запустіть

```bash
./open_files_test.sh
```

Якщо `ulimit -n` встановлено менше 100, скрипт виведе повідомлення про помилку.

---

## 3. Обмеження кількості процесів

### Перегляд ліміту

```bash
ulimit -u
```

### Встановлення обмеження

```bash
ulimit -u 50
```

### Перевірка

Спробуйте запустити багато процесів (наприклад, у циклі):

```bash
for i in {1..60}; do sleep 100 & done
```

Якщо ліміт 50, після 50-го процесу отримаєте помилку.

---

## 4. Обмеження часу CPU

### Перегляд

```bash
ulimit -t
```

### Встановлення обмеження (наприклад, 5 секунд)

```bash
ulimit -t 5
```

### Перевірка

Запустіть скрипт, який працює довше 5 секунд:

```bash
python3 -c "while True: pass"
```

Процес буде примусово завершений після 5 секунд.

---

# Важливо

- Зміни `ulimit` діють тільки для поточної сесії (терміналу).
- Для постійних змін потрібно редагувати `/etc/security/limits.conf` або конфігурації shell.
- Для підвищення hard limit потрібні права root.
