
## Опис скрипта `rsync.sh`

Цей Bash-скрипт автоматизує процес підготовки конфігураційних файлів, їх перетворення у формат JSON, копіювання на віддалений сервер та синхронізацію локальної папки з віддаленою за допомогою `rsync`.

---

### Основні функції скрипта:

1. **Визначення віддаленого шляху:**
   - Скрипт підключається до віддаленого сервера (ім'я сервера задається у змінній `REMOTE_HOST`) та отримує домашню директорію користувача.
   - На її основі формується шлях для деплою файлів (`REMOTE_PATH`).

2. **Генерація JSON-файлів з `.conf`:**
   - У директорії `CONFIG_DIR` скрипт шукає всі файли з розширенням `.conf`.
   - Для кожного файлу виконується функція `generate_function_json`, яка:
     - Зчитує пари ключ=значення з файлу.
     - Ігнорує порожні рядки та коментарі (рядки, що починаються з `#`).
     - Для змінної `LABELS` виконує парсинг у форматі `ключ=значення`, розділені комами, і формує JSON-об'єкт.
     - Формує JSON-об'єкт з усіма змінними та записує його у файл `function_<ім'я_конфігу>.json` у тій же директорії.

3. **Копіювання JSON-файлів на віддалений сервер:**
   - За допомогою `scp` копіюються всі згенеровані `function_*.json` файли у віддалену папку `${REMOTE_PATH}`.

4. **Перевірка синхронізації (dry-run):**
   - За допомогою `rsync` виконується пробна синхронізація (`--dry-run`) локальної директорії `WATCH_DIR` з віддаленою директорією `${REMOTE_PATH}/rsync/`.
   - Під час синхронізації виключаються каталоги `.git/` та `node_modules/`.

5. **Реальна синхронізація:**
   - Скрипт запитує у користувача підтвердження на виконання реальної синхронізації.
   - Якщо користувач вводить `y` або `Y`, виконується команда `rsync` для копіювання файлів.
   - Інакше деплой скасовується.

---

## Змінні, які можна налаштувати:

| Змінна       | Опис                                              |
|--------------|---------------------------------------------------|
| `CONFIG_DIR` | Локальна директорія з `.conf` файлами та JSON-файлами |
| `WATCH_DIR`  | Локальна директорія, вміст якої буде синхронізовано |
| `REMOTE_HOST`| Ім'я віддаленого сервера (повинен бути налаштований SSH config) |
| `REMOTE_HOME`| Автоматично отримується домашня директорія віддаленого користувача |
| `REMOTE_PATH`| Шлях на віддаленому сервері для копіювання файлів |

## Як запустити скрипт

### Крок 1. Підготовка

- Переконайтеся, що у вас налаштований доступ по SSH до віддаленого сервера під іменем `my-server` (або замініть `REMOTE_HOST` на актуальне ім'я/адресу).
- Переконайтеся, що у вас є права на запис у віддалену директорію `${REMOTE_HOME}/app`.
- Помістіть `.conf` файли у директорію, вказану у `CONFIG_DIR` (за замовчуванням `~/Documents/GOIT/rsync`).

### Крок 2. Надати права на виконання скрипта

```bash
chmod +x rsync.sh
```

## Приклад запуску скрипта

### Запуск у dry-run режимі (перевірка без фактичного копіювання)

Скрипт автоматично виконує dry-run під час запуску — це крок, коли `rsync` показує, які файли будуть синхронізовані, але нічого не копіює.

### Крок 3. Запустити скрипт

```bash
./rsync.sh
```

---

### Приклад виводу dry-run

```
Віддалена домашня директорія: /home/username
Віддалений шлях для деплою: /home/username/app
=============================================================
Згенеровано /home/username/Documents/GOIT/rsync/function_example.json:
{
  "KEY1": "value1",
  "LABELS": {
    "env": "prod",
    "version": "1.0"
  }
}

Копіюємо function_*.json на сервер...

--- Dry-run: Перевірка файлів для синхронізації ---
sending incremental file list
rsync/
file1.txt
file2.js

sent 150 bytes  received 45 bytes  390.00 bytes/sec
total size is 10240  speedup is 55.00

Виконати реальну синхронізацію? (y/N):
```

---

### Пояснення виводу dry-run

- Віддалена домашня директорія та шлях для деплою показані для підтвердження.
- Показано вміст згенерованого JSON-файлу.
- Після копіювання JSON-файлів на сервер виконується `rsync --dry-run` — він виводить список файлів, які будуть скопійовані.
- Наприкінці скрипт запитує у вас підтвердження, чи виконувати реальне копіювання.

---

### Запуск реальної синхронізації

Якщо ви введете `y` або `Y`, скрипт виконає реальну синхронізацію:

```
Виконуємо деплой...
sending incremental file list
rsync/
file1.txt
file2.js

sent 5000 bytes  received 230 bytes  15000.00 bytes/sec
total size is 10240  speedup is 1.8
Деплой завершено.
```

Якщо введете будь-що інше або просто натиснете Enter — деплой буде скасовано.


### Крок 4. Дочекайтеся виводу

- Скрипт виведе віддалену домашню директорію та шлях для деплою.
- Згенерує JSON-файли з `.conf`.
- Скопіює JSON-файли на сервер.
- Виконає dry-run синхронізації та покаже, які файли будуть скопійовані.
- Запитає, чи виконувати реальну синхронізацію.

### Крок 5. Підтвердіть або скасуйте деплой

- Введіть `y` або `Y` для підтвердження.
- Введіть будь-який інший символ або натисніть Enter для скасування.

---

## Додаткові поради

- Для автоматичного підключення по SSH без пароля рекомендується налаштувати SSH-ключі.
- Якщо потрібно змінити налаштування (шляхи, сервер), відредагуйте відповідні змінні у верхній частині скрипта.
- Скрипт ігнорує каталоги `.git/` та `node_modules/` під час синхронізації.
