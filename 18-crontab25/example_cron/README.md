# Аналіз вмісту `/etc/crontab`

```cron
17 *    * * *   root    cd / && run-parts --report /etc/cron.hourly
25 6    * * *   root    test -x /usr/sbin/anacron || { cd / && run-parts --report /etc/cron.daily; }
47 6    * * 7   root    test -x /usr/sbin/anacron || { cd / && run-parts --report /etc/cron.weekly; }
52 6    1 * *   root    test -x /usr/sbin/anacron || { cd / && run-parts --report /etc/cron.monthly; }
```

---

## Формат запису в `/etc/crontab`

У системному crontab формат має 6 полів:

```
<хвилина> <година> <день_місяця> <місяць> <день_тижня> <користувач> <команда>
```

- Перші 5 полів — це розклад (cron timing).
- 6 поле — ім’я користувача, від імені якого виконується команда.
- 7 поле — сама команда.

---

## Пояснення кожного рядка

---

### 1. Рядок

```cron
17 *    * * *   root    cd / && run-parts --report /etc/cron.hourly
```

- **Розклад:** `17 * * * *`

  - Хвилина: 17
  - Година: будь-яка (`*`)
  - День місяця: будь-який (`*`)
  - Місяць: будь-який (`*`)
  - День тижня: будь-який (`*`)

- **Користувач:** `root`

- **Команда:**

  ```bash
  cd / && run-parts --report /etc/cron.hourly
  ```

- **Що робить:**

  - Щогодини, на 17-й хвилині виконується команда.
  - `cd /` — переходить у кореневу директорію, щоб уникнути проблем з відносними шляхами.
  - `run-parts --report /etc/cron.hourly` — запускає всі скрипти, що знаходяться у директорії `/etc/cron.hourly`.
  - `--report` виводить інформацію про запущені скрипти.

- **Висновок:** Запускає всі скрипти з `/etc/cron.hourly` кожну годину о 17-й хвилині.

---

### 2. Рядок

```cron
25 6    * * *   root    test -x /usr/sbin/anacron || { cd / && run-parts --report /etc/cron.daily; }
```

- **Розклад:** `25 6 * * *`

  - Хвилина: 25
  - Година: 6 (06:25 ранку)
  - День місяця: будь-який
  - Місяць: будь-який
  - День тижня: будь-який

- **Користувач:** `root`

- **Команда:**

  ```bash
  test -x /usr/sbin/anacron || { cd / && run-parts --report /etc/cron.daily; }
  ```

- **Що робить:**

  - `test -x /usr/sbin/anacron` перевіряє, чи існує виконуваний файл `anacron`.
  - Якщо `anacron` **не встановлено** (`test -x` повертає false), виконується команда після `||`.
  - В цьому випадку запускаються всі скрипти у `/etc/cron.daily` (щоденні завдання).
  - Знову ж таки, переходить у `/` перед запуском.

- **Висновок:** Якщо `anacron` не встановлено, то щоденні завдання запускаються о 6:25 ранку.

---

### 3. Рядок

```cron
47 6    * * 7   root    test -x /usr/sbin/anacron || { cd / && run-parts --report /etc/cron.weekly; }
```

- **Розклад:** `47 6 * * 7`

  - Хвилина: 47
  - Година: 6
  - День місяця: будь-який
  - Місяць: будь-який
  - День тижня: 7 (неділя)

- **Користувач:** `root`

- **Команда:** аналогічна попередній, але запускає `/etc/cron.weekly`

- **Висновок:** Якщо `anacron` не встановлено, щотижневі завдання запускаються в неділю о 6:47 ранку.

---

### 4. Рядок

```cron
52 6    1 * *   root    test -x /usr/sbin/anacron || { cd / && run-parts --report /etc/cron.monthly; }
```

- **Розклад:** `52 6 1 * *`

  - Хвилина: 52
  - Година: 6
  - День місяця: 1 (перший день місяця)
  - Місяць: будь-який
  - День тижня: будь-який

- **Користувач:** `root`

- **Команда:** запускає `/etc/cron.monthly`, якщо `anacron` не встановлено.

- **Висновок:** Щомісячні завдання запускаються 1-го числа місяця о 6:52 ранку, якщо `anacron` відсутній.

---

## Що таке `anacron` і чому тут є перевірка?

- `anacron` — це утиліта, що виконує завдання cron, які могли бути пропущені, наприклад якщо сервер був вимкнений у час планового запуску.

- Якщо `anacron` встановлено і працює, він сам запускає щоденні, щотижневі і щомісячні завдання.

- Тому у системному crontab стоїть перевірка:

  - Якщо `anacron` є — ці завдання не запускаються тут (щоб не дублювати).
  - Якщо `anacron` немає — cron сам запускає відповідні скрипти.

---

## Що таке `run-parts`?

- `run-parts` — це утиліта, яка виконує всі скрипти (файли) у вказаній директорії.

- Вона запускає кожен скрипт послідовно.

- Опція `--report` виводить інформацію про те, які скрипти були запущені.
